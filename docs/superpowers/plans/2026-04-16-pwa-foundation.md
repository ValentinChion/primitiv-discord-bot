# PWA Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the existing `/schedule` page into an installable PWA that works offline, shows on the home screen, and prompts users to install it.

**Architecture:** Add a web app manifest via Next.js App Router's `app/manifest.ts`, register a service worker using `@serwist/next` (the maintained Workbox wrapper for Next.js), and add a client-side install prompt component to the schedule page. The service worker precaches all static assets and uses a stale-while-revalidate strategy for the `/api/schedule` endpoint so the schedule loads instantly even offline.

**Tech Stack:** Next.js 16 (App Router), `@serwist/next` v9, `serwist` v9, `sharp` (dev, one-time icon generation), Tailwind CSS v4

## Task 5: Write the service worker

**Files:**
- Create: `dashboard/app/sw.ts`

Caching strategy:
- **Precache**: all Next.js static assets (JS chunks, CSS) — serwist handles this via `__SW_MANIFEST`
- **`/api/schedule`**: `StaleWhileRevalidate` — show cached schedule instantly, refresh in background
- **Images (R2)**: `CacheFirst` with 30-day expiry — artist photos don't change
- **Navigation (HTML)**: `NetworkFirst` with offline fallback to `/offline`

- [ ] **Step 1: Create the service worker**

```ts
// dashboard/app/sw.ts
import type { PrecacheEntry, SerwistGlobalConfig } from "serwist";
import {
  CacheFirst,
  ExpirationPlugin,
  NetworkFirst,
  Serwist,
  StaleWhileRevalidate,
} from "serwist";

declare global {
  interface ServiceWorkerGlobalScope extends SerwistGlobalConfig {
    __SW_MANIFEST: (PrecacheEntry | string)[] | undefined;
  }
}

declare const self: ServiceWorkerGlobalScope;

const serwist = new Serwist({
  precacheEntries: self.__SW_MANIFEST,
  skipWaiting: true,
  clientsClaim: true,
  navigationPreload: true,
  runtimeCaching: [
    // Schedule API: serve stale, refresh in background
    {
      matcher: /\/api\/schedule/,
      handler: new StaleWhileRevalidate({
        cacheName: "schedule-api",
        plugins: [new ExpirationPlugin({ maxEntries: 1, maxAgeSeconds: 60 * 60 * 24 })],
      }),
    },
    // Artist images from R2: cache aggressively, they don't change
    {
      matcher: ({ request }) => request.destination === "image",
      handler: new CacheFirst({
        cacheName: "images",
        plugins: [
          new ExpirationPlugin({ maxEntries: 60, maxAgeSeconds: 60 * 60 * 24 * 30 }),
        ],
      }),
    },
    // Google Fonts: cache-first
    {
      matcher: /^https:\/\/fonts\.(googleapis|gstatic)\.com/,
      handler: new CacheFirst({
        cacheName: "google-fonts",
        plugins: [new ExpirationPlugin({ maxEntries: 10, maxAgeSeconds: 60 * 60 * 24 * 365 })],
      }),
    },
    // All navigation requests: try network, fall back to cached page or /offline
    {
      matcher: ({ request }) => request.mode === "navigate",
      handler: new NetworkFirst({
        cacheName: "pages",
        networkTimeoutSeconds: 5,
        plugins: [new ExpirationPlugin({ maxEntries: 10, maxAgeSeconds: 60 * 60 * 24 })],
      }),
    },
  ],
  fallbacks: {
    entries: [
      {
        url: "/offline",
        matcher({ request }) {
          return request.mode === "navigate";
        },
      },
    ],
  },
});

serwist.addEventListeners();
```

import { useEffect } from "react";

export function SwRegister() {
  useEffect(() => {

    window.addEventListener("load", () => {
      navigator.serviceWorker.register("/sw.js").catch(() => {
        // Registration failed silently — app still works, just without offline support
      });
    });
  }, []);

  return null;

## Task 8: Install prompt banner

**Files:**
- Create: `dashboard/components/install-prompt.tsx`
- Modify: `dashboard/app/schedule/page.tsx`

The `beforeinstallprompt` event fires on Android Chrome (and Chromium-based browsers) when the PWA criteria are met and the user hasn't installed it yet. On iOS, this event never fires — users must use Safari's Share → "Add to Home Screen" manually. The banner handles this gracefully: it only appears when the event fires (Android), and is dismissed permanently via `localStorage`.

- [ ] **Step 1: Create the install prompt component**

```tsx
// dashboard/components/install-prompt.tsx
"use client";

import { useEffect, useState } from "react";

interface BeforeInstallPromptEvent extends Event {
  prompt(): Promise<void>;
  userChoice: Promise<{ outcome: "accepted" | "dismissed" }>;
}

const DISMISSED_KEY = "ekotone-install-dismissed";

export function InstallPrompt() {
  const [prompt, setPrompt] = useState<BeforeInstallPromptEvent | null>(null);
  const [visible, setVisible] = useState(false);

  useEffect(() => {
    if (localStorage.getItem(DISMISSED_KEY)) return;

    const handler = (e: Event) => {
      e.preventDefault();
      setPrompt(e as BeforeInstallPromptEvent);
      setVisible(true);
    };

    window.addEventListener("beforeinstallprompt", handler);
    return () => window.removeEventListener("beforeinstallprompt", handler);
  }, []);

  const handleInstall = async () => {
    if (!prompt) return;
    await prompt.prompt();
    const { outcome } = await prompt.userChoice;
    if (outcome === "accepted") {
      setVisible(false);
    }
  };

  const handleDismiss = () => {
    localStorage.setItem(DISMISSED_KEY, "1");
    setVisible(false);
  };

  if (!visible) return null;

  return (
    <div className="fixed bottom-4 left-4 right-4 z-50 flex items-center justify-between gap-3 bg-[#161616] border border-[#2a2a2a] px-4 py-3 shadow-lg">
      <div className="min-w-0">
        <p className="font-mono-share text-[0.65rem] tracking-[0.15em] uppercase text-[#DDFF00] leading-tight">
          Installer l'appli
        </p>
        <p className="font-mono-share text-[0.58rem] tracking-[0.1em] uppercase text-[#555] mt-0.5">
          Accès hors-ligne · Écran d'accueil
        </p>
      </div>
      <div className="flex items-center gap-2 shrink-0">
        <button
          onClick={handleInstall}
          className="font-mono-share text-[0.6rem] tracking-[0.18em] uppercase bg-[#DDFF00] text-[#0F0F0F] px-3 py-1.5 cursor-pointer"
        >
          Installer
        </button>
        <button
          onClick={handleDismiss}
          aria-label="Fermer"
          className="font-mono-share text-[0.7rem] text-[#444] hover:text-[#888] cursor-pointer bg-transparent border-none px-1"
        >
          ×
        </button>
      </div>
    </div>
  );
}
```

## Self-review notes

- Serwist is disabled in `development` mode intentionally. If you want to test SW behaviour locally, temporarily remove the `disable` line in `next.config.ts` and hard-reload.
- The `scripts/generate-icons.mjs` script is a one-time tool. After running it, the PNGs are committed and the script can stay in the repo for re-generation later (e.g. when brand assets change).
- iOS users won't see the install banner (no `beforeinstallprompt`). A future improvement could be a separate "add to home screen" hint for iOS Safari users (detect via `navigator.userAgent`), but that's out of scope here.
- `public/sw.js` is generated at build time. It should be added to `.gitignore` or committed — since it's generated, prefer `.gitignore`.
